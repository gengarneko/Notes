app()->getLocale() 获取的是 config/app.php 的 locale 选项, 因为我们之前在基础配置中进行了修改, 所以这里的值应该为 zh-CN

<meta name="csrf-token" content="{{ csrf_token() }}">

csrf-token 标签是为了方便前端的 js 脚本获取 CSRF 令牌

@yield('title', 'RepairSystem') 继承此模板的页面, 如果没有定制 title 区域的话, 自动使用第二个参数 

asset('css/app.css') 使用当前的请求协议 HTTP/HTTPS 为资源文件生成一个 URL, 以我们的情况, 会生成  http://http://repairsystem.test/css/app.css

<div id="app" class="{{ route_class() }}-page">

route_class() 方法是我们自定义的辅助方法, 在 helpers.php 中添加此方法:

bootstrap/helpers.php

<?php

function route_class()
{
    return str_replace('.', '-', Route::currentRouteName());
}

该辅助方法将当前请求的路由名称转换为 CSS 类名称, 作用是允许我们针对某个页面做页面样式定制, 在后面会提到

首页展示

1.创建控制器

使用 PagesController 来管理所有自定义页面的逻辑, 并使用 root() 方法来处理首页的展示

$ php artisan make:controller PagesController

我们在文件中新增 root() 方法

app/Http/Controllers/PagesController.php

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class PagesController extends Controller
{
    public function root()
    {
        return view('pages.root');
    }
}

2.视图

控制器中的 root() 方法加载了视图 pages.root, 但是目前我们还没有该页面, 我们在 views 中创建

resources/views/pages/root.blade.php

@extends('layouts.app')
@section('title', '首页')

@section('content')
  <h1>这里是首页</h1>
@stop

我们将自带的 welcome 视图删除

$ rm resources/views/welcome.blade.php

3.绑定路由

接下来绑定路由, 将 web.php 里的内容替换为

routes/web.php

<?php

Route::get('/', 'PagesController@root')->name('root');
