3.5--用户注册页面

静态页面

后面将会添加注册登录的功能,现在先让我们完成基本静态页面的编写

注册路由

routes/web.php

<?php

Route::get('/', 'StaticPagesController@home')->name('home');
Route::get('/help', 'StaticPagesController@help')->name('help');
Route::get('/about', 'StaticPagesController@about')->name('about');

Route::get('signup', 'UsersController@create')->name('signup');

我们将 signup 登录请求交给用户控制器 UserController 来处理
后面会为注册页面加上表单注册功能,到时候表单请求会和数据库进行交互,因此该页面不算静态页面,也就不能通过 StaticPagesController 进行处理 

signup 和 /signup 两种写法 Laravel 都兼容

生成用户控制器

php artisan make:controller UsersController

路由中,将用户注册页面的请求处理指定给用户控制器的create方法进行处理, 定义该控制器的 create 方法

app/Http/Controllers/UsersController.php

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use App\Http\Requests;

class UsersController extends Controller
{
    public function create()
    {
        return view('users.create');
    }
}
