基本模型

首先我们要先为微博创建一个数据表来存储微博数据, 一条简单的微博至少拥有微博内容(content) 和微博发布者 (user_id) 的休息, 下面让我创建一个新的迁移文件来创建微博数据表

$ php artisan make:migration create_statuses_table --create="statuses"

这里我们把微博表明设置为 statuses 意为 动态

打开新创建的微博迁移文件并加入以下内容

database/migrations/[timestamp]_create_statuses_table.php

第一个字段用于存储微博的内容
第二个字段用于存储微博发布者的个人 id, 为了提高效率, 加上索引
第三个则为微博的创建时间添加索引
, 由于 timestamps 方法会为微博生成一个创建时间字段和一个更新字段, 因此我们不需要再次手动创建 created_at 字段, 为创建时间添加索引的目的是, 后面会根据微博创建时间进行倒序输出,


在我们定义好迁移文件之后, 运行以下命令来执行迁移

$ php artisan migrate

\接着我们还需要创建微博模型来跟数据库进行交互, 新建的模型文件统一放在 app/Models 文件夹下面, 为此我们在新建一个新的模型对象的时候, 需要在模型名称前面加上 Models 目录

$ php artisan make:model Models/Status



用户和微博之间的关联

用户和微博之间是一对多的关系

如果没有一对多的关系, 我们需要这样来创建一条微博

App\Models\Status::create()

当我们将用户模型和微博模型进行一对多关联之后, 得到以下方法

$user->statuses()->create()

这样在微博进行创建时便会自动关联微博与用户之间的关系, 非常方便

Eloquent 模型让关联的管理和处理变得更加简单, 同时也支持以下几种关联类型:
  1. 一对一
  2. 一对多
  3. 多对多
  4. 远层一对多
  5. 多态关联
  6. 多态多对多关联

我们先从一对多学习模型对应关系, 下一章在构建用户粉丝关系的时候, 我们会学习多对多关联, 我们可以在模型中将 Eloquent 关联定义为函数,, 如下, 指明一条微博属于一个用户

app/Models/Status.php

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Status extends Model
{
    public function user()
    {
        return $this->belongsTo(User::class);
    }
}
