5.4--用户数据验证

用户数据验证

如果现在填写注册表单进行提交, 会报错, 因为表单还不能正常的使用, 我们还需要在用户控制器中添加一个用于处理表单数据提交后的 store 方法, 用于处理用户创建的相关逻辑

app/Http/Controllers/UsersController.php

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use App\Http\Requests;
use App\Models\User;

class UsersController extends Controller
{
    public function create()
    {
        return view('users.create');
    }

    public function show(User $user)
    {
        return view('users.show', compact('user'));
    }

    public function store(Request $request)
    {
        $this->validate($request, [
            'name' => 'required|max:50',
            'email' => 'required|email|unique:users|max:255',
            'password' => 'required|confirmed|min:6'
        ]);
        return;
    }
}

在实际开发中, 我们需要对用户输入的数据进行验证, 在验证成功后再将数据存入数据库, 在 Laravel 中, 提供了多种数据验证方式, 在本教程中, 我们使用其中一种对新手比较友好的方式来进行验证. -- validator. validator 由 App\Http\Controllers\Controller 类中的 ValidatesRequest 进行定义, 因此我们可以在所哟的控制器中使用 validate 种方法来进行数据验证, validate 方法接受两个参数, 一个是用户的输入数据, 第二个是参数为该输入数的验证规则

验证规则

存在性验证

'name'  => 'required',

长度验证

'name' => 'min:3|max:6',

格式验证

'email' => 'email',

唯一性验证

'password' => 'unique:users'

但是这种方法并不严谨, 我们需要在一开始创建用户数据时便设置邮箱字段的唯一属性. 这个 Laravel在默认我们生成的用户表迁移中便已经设定好了.

密码匹配原则

'password' => 'confirmed'

表单提交

在添加好验证规则之后, 尝试点击会报错 页面过期

出现原因是, 在我们使用 POST 方法提交表单时, Laravel 安全考虑, 会让我们提供一个 token 令牌, 来防止我们应用受到 CSRF(跨站请求伪造) 的攻击, 修复异常的方法很简单, 我们只需要在表单元素中添加 Blade 模板为我们提供的 csrf_field 方法即可, 该方法在 blade 模板中调用如下:

{{ csrf_field() }}

上面这段代码转化如下:

<input type="hidden" name="_token" value="fhcxqT67dNowMoWsAHGGPJOAWJn8x5R5ctSwZrAq">

由于输入框是 hidden 类型, 因此该 input 元素在页面上面是不可见的. 现在让我们为注册表单添加 csrf_field 方法.

resources/views/users/create.blade.php

@extends('layouts.default')
@section('title', '注册')

@section('content')
<div class="col-md-offset-2 col-md-8">
  <div class="panel panel-default">
    <div class="panel-heading">
      <h5>注册</h5>
    </div>
    <div class="panel-body">
      <form method="POST" action="{{ route('users.store') }}">
          {{ csrf_field() }}

          <div class="form-group">
            <label for="name">名称：</label>
            <input type="text" name="name" class="form-control" value="{{ old('name') }}">
          </div>

          <div class="form-group">
            <label for="email">邮箱：</label>
            <input type="text" name="email" class="form-control" value="{{ old('email') }}">
          </div>

          <div class="form-group">
            <label for="password">密码：</label>
            <input type="password" name="password" class="form-control" value="{{ old('password') }}">
          </div>

          <div class="form-group">
            <label for="password_confirmation">确认密码：</label>
            <input type="password" name="password_confirmation" class="form-control" value="{{ old('password_confirmation') }}">
          </div>

          <button type="submit" class="btn btn-primary">注册</button>
      </form>
    </div>
  </div>
</div>
@stop


如果再次填写表单并进行提交, 会发现表单依旧处于不可提交的状态. 这是因为:

  1. 在表单信息验证失败的时候, 页面上没有给出错误提示.
  2. 在表单信息验证通过的时候, 页面没有重定向到其他页面, 并给出注册成功的提示.
